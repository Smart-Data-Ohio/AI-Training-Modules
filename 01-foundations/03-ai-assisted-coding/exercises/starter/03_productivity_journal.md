# Exercise 03: Productivity Journal

## The Goal

Use an AI coding tool for a real task and honestly track whether it made you faster, slower, or about the same.

Most claims about AI coding productivity are either marketing ("10x faster!") or backlash ("it makes you worse!"). This exercise gives you your own data.

## Instructions

### Step 1: Pick a Real Task

Choose a coding task you'd normally do this week. It should be:
- Something you could complete in 30-60 minutes without AI
- Real work, not a toy problem
- Something you know how to do (so you can judge the AI's output)

Good examples:
- Write a new API endpoint with validation
- Add unit tests for an existing module
- Refactor a component to use a new pattern
- Build a data transformation script
- Write a database migration

### Step 2: Work with AI, Journal as You Go

Use your preferred AI coding tool for the task. As you work, fill in the log below **in real time** — don't try to reconstruct it from memory afterward.
